## 🏆 Engineering Highlights

<details>
<summary><b>⚡ 95% Server Refresh Performance Gain — ACM Platform</b></summary>
<br/>

**Problem:** Server health monitoring across 200+ production servers took 8–10 minutes per refresh cycle — too slow for real-time operations.

**Solution:** Redesigned the entire status pipeline using:
- Parallel `async/await` processing with `ConcurrentDictionary` for thread-safe state management
- Independent health detection decoupled from the central TMDC agent
- Background processing for non-blocking status updates

**Result:** Refresh time dropped to **under 30 seconds** — enabling near real-time monitoring.

**Tech:** ASP.NET Core · Angular · MongoDB · REST APIs · Async Programming

</details>

<details>
<summary><b>🚀 Zero-Downtime .NET 8 Migration — DIGenx Platform</b></summary>
<br/>

**Problem:** Legacy enterprise app stuck on .NET Core 2.1 (end-of-life) — performance bottlenecks and security risks across 15+ modules.

**Solution:**
- Migrated 15+ controllers with zero production incidents
- Optimized EF Core queries across 30+ database tables
- Built dynamic PDF generation engine for client-facing sales quotes
- Designed event-driven email notification system for CRUD operations on sales opportunities

**Result:** **94% faster page loads**, eliminated EOL dependency risks, and automated ~200 quotes/month.

**Tech:** ASP.NET Core (.NET 8) · Angular · MySQL · Entity Framework Core · REST APIs

</details>

<details>
<summary><b>⚙️ Event-Driven Test Automation System</b></summary>
<br/>

**Problem:** Manual test-case uploads couldn't scale with the growing QA pipeline.

**Solution:** Built a distributed mass test-case upload system with:
- **RabbitMQ** for async message-driven communication
- Decoupled architecture: Web API → Job Scheduler → Upload Engine
- Reliable retry and dead-letter queue handling

**Result:** Fully automated, scalable test-case ingestion pipeline.

**Tech:** ASP.NET Core · RabbitMQ · Web API · Background Services

</details>
